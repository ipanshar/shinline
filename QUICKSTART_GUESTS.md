## 🎯 Функциональность

### 📋 Основные операции

1. **Создание гостевого визита**
   - ФИО гостя, ИИН, компания, должность, телефон
   - Встречающая сторона и её телефон  
   - Период визита (начало и конец)
   - Тип пропуска (разовый или многоразовый)
   - Комментарий/цель визита
   - Опциональное добавление ТС

2. **Управление ТС гостя**
   - Добавить одно или несколько ТС
   - Гос. номер (обязательное)
   - Марка, модель, цвет (опциональные)
   - Удаление ТС из визита

3. **Управление статусами**
   - Активный → Закрытый (выпуск пропусков)
   - Активный → Отменённый (отмена визита)
   - Отметить приход гостя (check-in)
   - Отметить уход гостя (check-out)

4. **Управление пропусками**
   - Выпуск пропусков (person + vehicle)
   - Отзыв пропусков
   - Синхронизация с DSS (для транспортных)

5. **Поиск и фильтрация**
   - Поиск по: ФИО, ИИН, компании, встречающему
   - Фильтры: двор, статус, тип пропуска, наличие ТС, дата

## 🔑 Требуемые права доступа

Нужна одна из этих ролей:
- **Администратор**
- **Оператор** (рекомендуется)
- **Управляющий**

Или минимум permission:
- `guest_visits.view` (просмотр)

Остальные permissions (по необходимости):
- `guest_visits.create` (создание)
- `guest_visits.update` (редактирование)
- `guest_visits.close` (закрытие)
- `guest_visits.cancel` (отмена)
- `guest_visits.issue_permits` (выпуск пропусков)

## 📝 Примеры использования

### Сценарий 1: Одноразовый визит гостя без ТС

1. Нажмите "+ Новый гостевой визит"
2. Заполните:
   - Двор: выберите из списка
   - ФИО гостя: "Иван Петров"
   - ИИН: "123456789012" (опционально)
   - Должность: "Менеджер"
   - Телефон гостя: "+77777777777"
   - Встречающий: "Алексей"
   - Телефон встречающего: "+78888888888"
   - Время начала: выберите время
   - Цель: "Деловая встреча"
   - Тип пропуска: "Разовый"
3. Нажмите "Создать"
4. При приходе: кнопка LogIn (зелёная) → "Приход отмечен"
5. При уходе: кнопка LogOut (оранжевая) → "Уход отмечен" → визит автоматически закроется

### Сценарий 2: Визит с ТС

1. Нажмите "+ Новый гостевой визит"
2. Заполните основные поля (как выше)
3. В блоке "Транспорт гостя":
   - Нажмите "+ Добавить ТС"
   - Гос. номер: "A001BC"
   - Марка: "Toyota"
   - Модель: "Camry"
   - Цвет: "Белый"
4. Нажмите "Создать"
5. При проезде: создастся пропуск на ТС и синхронизируется с DSS

### Сценарий 3: Многоразовый визит

1. Нажмите "+ Новый гостевой визит"
2. Заполните основные поля
3. **На вкладке "Тип пропуска"**: выберите "Многоразовый"
   - Обязательно укажите "Время окончания визита"
4. Нажмите "Создать"
5. Пропуска не будут деактивироваться после первого выхода
6. Станут неактивными после visit_ends_at или при ручной отмене

## 🛠️ Техническая информация

### Структура данных в БД

**guest_visits**
```
- id: PK
- yard_id: FK
- guest_full_name, guest_iin, guest_company_name, guest_position
- guest_phone, host_name, host_phone
- visit_starts_at, visit_ends_at
- permit_kind: enum(one_time, multi_time)
- workflow_status: enum(active, closed, canceled)
- has_vehicle: boolean
- comment: text
- last_entry_at, last_exit_at, closed_at: timestamps
- source: enum(operator, integration, import, telegram_bot)
- created_by_user_id, approved_by_user_id, cancelled_by_user_id: FKs
- created_at, updated_at
```

**guest_visit_vehicles**
```
- id: PK
- guest_visit_id: FK
- truck_id: FK (nullable)
- plate_number, brand, model, color, comment
- created_at, updated_at
```

**guest_visit_permits** (orchestration)
```
- id: PK
- guest_visit_id: FK
- entry_permit_id: FK
- permit_subject_type: enum(person, vehicle)
- guest_visit_vehicle_id: FK (nullable)
- revoked_at: timestamp (nullable)
- created_at
```

**visitors** (extended)
```
- ... (existing fields)
- guest_visit_id: FK (nullable) ← NEW
- comment: text (extended)
```

## 🔄 Интеграция с существующими системами

### DSS (Система декоративного контроля)
- ✅ Транспортные пропуска GuestVisit синхронизируются с DSS
- ✅ Используется существующий `DssPermitVehicleService`
- ✅ Полностью не влияет на основной DSS контур

### Система пропусков (EntryPermit)
- ✅ GuestVisit оркестрирует EntryPermit
- ✅ Не заменяет, а дополняет EntryPermit
- ✅ Поддержка миграции из старой системы

### Система посетителей (Visitor)
- ✅ Visitor связан с GuestVisit через `guest_visit_id`
- ✅ Полная трассировка: заявка → пропуск → визит
